# Repo instructions — always loaded

This repo is a Claude Code Routine. Everything you write ends up in a public-facing daily brief.

## Voice
- Direct, analytical, newsletter-quality. No hype.
- One clear thought per sentence.
- Never write "revolutionary", "game-changing", "cutting-edge", or "unleash".

## Formatting
- Markdown only. No HTML.
- Code blocks for commands, `backticks` for repo names.
- Link syntax: `[repo/name](https://github.com/repo/name)`.

## Never
- Modify or delete existing files in `outputs/`.
- Invent a star count or a date.
- Feature a repo you cannot verify exists.
- Push anything outside the `outputs/` folder.

## Always
- UTC dates in filenames.
- Include the status note when partial.
- Dedupe against `outputs/` last-3-days.
